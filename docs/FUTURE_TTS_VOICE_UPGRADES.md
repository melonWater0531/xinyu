# Future TTS Voice Upgrades

> 当前版本只落地浏览器 Web Speech 初版：后端广播 voice event，前端负责朗读和降级显示。
> 本文记录暂未执行的完整语音能力，便于后续继续升级部署。

## 1. 云端 TTS Provider 路由与缓存

- 暂缓原因：当前目标是不新增 API key 和后端音频依赖。
- 依赖：TTS provider 选择、鉴权、计费策略、音频缓存目录、失败 fallback。
- 建议路径：新增 `services/tts_router.py`，复用 LLM router 的 provider 顺序思想，返回音频 URL 或浏览器朗读 fallback。

## 2. 主机扬声器或 ReSpeaker 输出

- 暂缓原因：WSL/主机音频输出设备差异大，可能和 ReSpeaker 输入、会议录音抢占设备。
- 依赖：输出设备枚举、音量控制、播放队列、会议录音时的回声抑制。
- 建议路径：保持浏览器播放为默认；新增可选 `TTS_OUTPUT_DEVICE` 后再允许主机侧播放。

## 3. 流式 TTS

- 暂缓原因：Web Speech 初版不需要服务端流式音频。
- 依赖：流式 TTS provider、分句策略、前端缓冲和取消播放。
- 建议路径：先按句子切分 LLM 输出，再升级为边生成边播放。

## 4. 语音输入 ASR + 多轮语音对话

- 暂缓原因：当前只实现语音输出，实时语音输入会引入 VAD、ASR、打断和隐私授权。
- 依赖：wake word 后录音窗口、云端/本地 ASR、对话状态机、超时退出。
- 建议路径：使用 wake word 触发短录音，再复用 `/api/chat`，最终形成 hands-free 多轮模式。

## 5. 小屿固定声线与情绪化音色

- 暂缓原因：浏览器内置音色不可完全固定。
- 依赖：云端 TTS 或本地声线模型、用户授权、音色参数调优。
- 建议路径：保留当前 `reason/priority` 元数据，后续映射到 voice/style/speed。

## 6. 更完整的主动陪伴策略

- 暂缓原因：初版只做克制提醒和冷却，避免打扰用户。
- 依赖：用户偏好、安静时段、提醒历史、更多状态判定样本。
- 建议路径：把提醒策略从前端本地通知逐步迁到 `services/voice_policy.py`，并增加可解释 debug trace。

## 7. 会议语音播报与说话人确认

- 暂缓原因：会议录音中频繁播报会干扰录音和参会人。
- 依赖：回声抑制、会议模式权限、说话人命名 UI。
- 建议路径：默认只播开始/结束/摘要完成；后续允许用户手动朗读摘要或确认说话人。

## 8. 隐私与敏感内容不朗读

- 暂缓原因：初版只朗读短提示和聊天回复，不做复杂敏感内容分类。
- 依赖：敏感词/场景策略、用户静音设置、前端语音历史清除。
- 建议路径：新增 `can_speak_text()` 策略，对日记正文、会议原文和敏感健康内容默认不朗读。

## 9. 前端语音历史与诊断面板

- 暂缓原因：当前 `/control` 已显示最近 voice events，产品页只保留轻量状态。
- 依赖：播放队列、失败原因、浏览器 voice 列表。
- 建议路径：在 `/control` 增加浏览器回报事件，如 `voice_play_started/ended/error`，形成端到端诊断。
